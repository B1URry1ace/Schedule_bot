# 05. Компонентная диаграмма


### Telegram Bot API
Точка входа для всех сообщений, команд и callback-кнопок.

### `handlers.py`
Обрабатывает:
- команды `/start`, `/today`, `/tomorrow`, `/week`, `/schedule`;
- выбор дат по кнопкам;
- добавление заметок;
- FSM-сценарии.

### `services.py`
Содержит бизнес-логику:
- резолв группы в `.ics`-ссылку;
- загрузку и парсинг календаря;
- распознавание голосовой заметки.

### `database.py`
Хранит:
- привязку пользователя к группе;
- заметки к событиям.

### `keyboards.py`
Генерирует inline-клавиатуры для навигации.

### `states.py`
Хранит FSM-состояния:
- ожидание ввода группы;
- ожидание заметки.

### `config.py`
Хранит конфигурацию приложения:
- токен бота;
- таймзону;
- параметры LM Studio;
- модель Whisper;
- список групп и идентификаторов.

## Потоки взаимодействия

### Поток 1: просмотр расписания
Пользователь - Telegram Bot API - `handlers.py` - `database.py` - `services.py` - внешний `.ics` - обратно в `handlers.py` - сообщение в Telegram

### Поток 2: добавление заметки
Пользователь - `handlers.py` - FSM - `database.py` - Telegram Bot API

### Поток 3: голосовая заметка
Пользователь - Telegram Bot API - `handlers.py` - скачивание файла - `services.py` - Whisper → LM Studio - `database.py` - Telegram Bot API
