# 06. Последовательность работы бота


## 1. Пользователь вводит группу, а бот показывает расписание

Сначала пользователь отправляет команду `/start`. Бот отвечает сообщением с просьбой написать номер группы или прислать ссылку на `.ics`-календарь.

После этого пользователь вводит номер группы, например `КТбо3-3`. Обработчик получает это сообщение и передаёт его в сервисный слой. Там происходит поиск нужной `.ics`-ссылки для этой группы.

Когда ссылка найдена, бот сохраняет данные пользователя в базе: идентификатор пользователя, название группы и ссылку на календарь. После сохранения бот сообщает, что группа успешно добавлена, и показывает меню.

Если пользователь вводит команду `/today`, бот сначала берёт сохранённую для него `.ics`-ссылку из базы данных. Затем он обращается к внешнему календарю и получает события на сегодня. После этого бот отправляет пользователю расписание на текущий день.

## 2. Пользователь добавляет заметку к паре

Когда пользователь нажимает кнопку для добавления или изменения заметки, бот переводит диалог в режим ожидания заметки.

После этого бот просит отправить текст или голосовое сообщение.

Если пользователь отправляет обычный текст, бот использует его сразу. Если приходит голосовое сообщение, бот скачивает его и распознаёт с помощью системы распознавания речи. Затем текст при необходимости нормализуется.

После этого бот сохраняет заметку в базе данных, привязывая её к конкретной паре и пользователю. Когда сохранение завершено, бот сообщает, что заметка успешно добавлена.

## Общая логика работы

В работе бота можно выделить три основных части:

- `handlers.py` отвечает за команды, сообщения и переходы между состояниями;
- `services.py` получает данные из внешних источников и обрабатывает их;
- `database.py` хранит информацию о пользователях и заметках.
